# Output Format Support

SandAgent runner-cli now supports multiple output formats via the `--output-format` option.

## Usage

```bash
sandagent run [options] --output-format <format> -- "<user input>"
```

### Short Option

```bash
sandagent run -o <format> -- "<user input>"
```

## Supported Formats

### 1. Stream Format (Default)

The default format outputs Server-Sent Events (SSE) using the [AI SDK UI Stream Protocol](https://ai-sdk.dev/docs/ai-sdk-ui/stream-protocol).

**Use case:** Real-time UI streaming, compatible with AI SDK UI components

**Example:**
```bash
sandagent run -- "Calculate 2+2"
```

**Output:**
```
data: {"type":"start","messageId":"msg_123"}
data: {"type":"text-start","id":"text_1"}
data: {"type":"text-delta","id":"text_1","delta":"The answer is 4."}
data: {"type":"text-end","id":"text_1"}
data: {"type":"finish","finishReason":"stop"}
data: [DONE]
```

### 2. JSON Format

Outputs a structured JSON object with complete message content and metadata.

**Use case:** API integration, automation, result parsing

**Example:**
```bash
sandagent run --output-format json -- "Calculate 2+2"
# or
sandagent run -o json -- "Calculate 2+2"
```

**Output:**
```json
{
  "content": [
    {
      "type": "text",
      "text": "The answer is 4."
    }
  ],
  "metadata": {
    "messageId": "msg_123",
    "model": "claude-sonnet-4-20250514",
    "sessionId": "session_456",
    "finishReason": "stop",
    "usage": {
      "promptTokens": 10,
      "completionTokens": 8
    }
  }
}
```

## JSON Output Structure

The JSON output contains three main sections:

### 1. Content Array

An array of content blocks representing the agent's response:

```typescript
{
  content: Array<{
    type: "text" | "tool-call" | "tool-result";
    text?: string;              // For text content
    toolCall?: {                // For tool calls
      id: string;
      name: string;
      input: unknown;
    };
    toolResult?: {              // For tool results
      id: string;
      result: unknown;
    };
  }>;
}
```

### 2. Metadata Object

Contains message and execution metadata:

```typescript
{
  metadata?: {
    messageId?: string;         // Unique message identifier
    tools?: unknown[];          // Available tools list
    model?: string;             // Model used
    sessionId?: string;         // Session identifier
    finishReason?: string;      // Completion reason
    usage?: {                   // Token usage
      promptTokens?: number;
      completionTokens?: number;
    };
  };
}
```

### 3. Error Field (Optional)

If an error occurred during execution:

```typescript
{
  error?: string;               // Error message
}
```

## Examples

### Basic Usage

```bash
# Stream format (default)
sandagent run -- "Write a hello world script"

# JSON format
sandagent run --output-format json -- "Write a hello world script"
```

### With Other Options

```bash
# JSON format with specific model
sandagent run -o json -m claude-sonnet-4-20250514 -- "Analyze data.csv"

# JSON format with template
sandagent run -o json --template coder -- "Build a REST API"

# JSON format with max turns
sandagent run -o json --max-turns 10 -- "Debug my code"
```

### Automation Example

```bash
# Save JSON output to file
sandagent run -o json -- "Calculate pi to 10 digits" > result.json

# Parse JSON with jq
sandagent run -o json -- "What is 2+2?" | jq '.content[0].text'

# Use in scripts
result=$(sandagent run -o json -- "Generate UUID")
uuid=$(echo "$result" | jq -r '.content[0].text')
echo "Generated UUID: $uuid"
```

## Implementation Details

The JSON format is generated by:

1. Running the agent and collecting the SSE stream
2. Parsing each SSE event line
3. Aggregating text deltas into complete text blocks
4. Collecting tool calls and results
5. Extracting metadata from stream events
6. Outputting a single structured JSON object

This ensures compatibility with the AI SDK UI protocol while providing a convenient structured format for automation.
